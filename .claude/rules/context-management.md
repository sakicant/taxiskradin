# Context Management Rules

- Save important decisions, findings, and progress to files — don't rely on conversation memory alone
- When a session involves significant work, periodically update context files
- At the end of complex tasks, write a summary to the appropriate context file
- Reference context files rather than asking the user to repeat information
- When context is getting long, proactively summarize and save before continuing
- Keep CLAUDE.md updated when workspace structure or conventions change
- Use task lists for multi-step work to maintain progress across context windows
- If you notice information might be lost to context compression, save it immediately

**Best practices:**
- Start sessions by reading relevant context files
- Don't assume information from earlier in a long conversation is still available
- Write to context/ for persistent information, not just conversation memory
